使用者針對「靈媒遊戲」(Phantom Ink) 的 AI 題目生成系統，提出了嚴格的三階段流程設計：

1. **出題階段 (AI A)** — 扮演出題老師，根據謎底生成七道由難到易的問答，有嚴格規則約束
2. **驗題階段 (AI Reviewer)** — 扮演驗題老師，檢查暴雷、難度遞增、重複、歧義等
3. **模擬玩家階段 (AI B)** — 扮演玩家，只能看到已揭露的題目與有限注音，記錄在哪一題猜出

**核心約束：**
- 謎底必須是具體名詞
- 七題由難到易，前三題不能讓一般人猜出
- 第七題必須幾乎可以猜出
- 回答必須只有一句短語，不超過六個中文字
- 回答不能直接包含謎底
- 回答必須可轉注音、唯一合理

**Why:** 避免 LLM 直接生成七題時產生暴雷、重複、無層次等品質問題
**How to apply:** 所有實作必須遵循生成→驗證→模擬的三階段流程，且每個階段有不同的 prompt 約束
